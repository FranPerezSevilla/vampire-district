# Masquerade Systems Plan

## Design intent

Eliminating an NPC is not always the same action. The player must be able to stun, kill, or drain civilians, police, hunters and the journalist. These actions create different kinds of pressure.

## Milestone 1 - Neutralization verbs

Add three close-range verbs:

- Stun: temporary non-lethal control. The target stays visible, cannot move, cannot report, and recovers after a few seconds.
- Kill: lethal elimination without feeding. Leaves a body and normal evidence pressure.
- Drain: vampiric feeding. Reduces hunger and leaves stronger supernatural evidence.

## Milestone 2 - Masquerade breach rules

Only draining should create masquerade panic. Public powers such as Shadow Dash / smoke or Whisper should not automatically break stealth or alert witnesses.

When a drain is witnessed:

1. The witness freezes for a short WTF reaction window.
2. The witness then flees to a report point.
3. If they reach the report point, the masquerade is broken and the mission fails.
4. The report forces maximum exposure, which brings police and hunters.

## Milestone 3 - Public violence distinction

Public stun / kill can still create mundane suspicion, but it is not a masquerade breach by itself. It can raise exposure and create body cleanup pressure, but hunters should not appear just because a mundane body exists.

## Milestone 4 - NPC movement readability

NPCs should not collide with each other. Police, hunters, witnesses and lured NPCs should navigate around buildings using street navigation points instead of pushing directly into walls.

## Milestone 5 - Feedback

The UI should clearly indicate:

- STUNNED targets.
- Lured targets.
- Witnesses frozen in WTF state.
- Witnesses fleeing to report.
- Masquerade failure state.
